HTTP (Hypertext Transfer Protocol) Method란
- 클라이언트와 서버 간의 통신에서 수행하려는 작업을 정의하는 데 사용되는 명령임.
- HTTP 요청은 클라이언트가 서버에 ★★★어떤 작업★★★을 요청하는지를 명확히 하기 위해 특정 HTTP 메서드를 포함하고 있다.


1. GET: 서버에서 데이터를 가져오거나 정보를 요청시 사용하는 메소드. 서버의 상태를 변경하지 않음.
예시: 웹 페이지를 요청하는 경우 사용 (select)


2. POST: 서버에 데이터를 전송하여 새로운 리소스를 생성하거나 기존 리소스를 업데이트할 때 사용하는 메소드. 주로 폼 제출이나 파일 업로드에 사용됩니다. (insert)
예시: 회원가입 폼을 제출할 때 사용

3. PUT: 특정 리소스의 전체 데이터를 수정(갱신)하는 데 사용하는 메서드 (where조건이 없는 update)
예시: 기존 사용자의 모든 정보를 업데이트할 때 사용
(단, 파일데이터의 자원을 전송하는 경우는 POST를 사용하는 것이 좋다. PUT으로는 전송 불가)

4. DELETE: 서버에서 지정된 리소스를 삭제하는 데 사용하는 메서드(delete)
예시: 사용자를 삭제할 때 DELETE 메서드를 사용

5. PATCH: 기존 리소스를 부분적으로 업데이트하는 경우 사용하는 메서드(delete)
PUT과 달리 전체 리소스를 수정하는 것이 아니라, 일부 속성만 수정함.
예시: 사용자 정보 중 일부만 수정할 때 PATCH 메서드를 사용

6. HEAD: 리소스의 메타데이터(예: 수정 날짜, 콘텐츠 길이 등)를 확인할 때 사용하는 메서드(delete)
GET 메서드와 비슷한듯 하지만, 응답 본문(body)은 포함되지 않고 헤더 정보만 포함됨. 
예시: 페이지의 헤더 정보를 확인할 때 HEAD 메서드가 사용됩니다.

그 외에 OPTIONS, CONNECT, TRACE등의 메소드도 존재함.
